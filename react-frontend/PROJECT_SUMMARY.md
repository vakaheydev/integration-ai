# OpenAPI Analyzer - Краткое описание проекта

## Общая информация

**Название:** OpenAPI Analyzer Frontend  
**Тип проекта:** Дипломный проект (Frontend SPA)  
**Технологии:** React 18 + TypeScript + Vite + Material UI + Axios  
**Статус:** ✅ Готово к использованию

---

## Назначение

Одностраничное веб-приложение (SPA) для взаимодействия с backend-системой анализа OpenAPI (Swagger) документов с использованием LLM. Позволяет загружать документы, просматривать их и задавать вопросы, получая интеллектуальные ответы от AI модели.

---

## Ключевые возможности

### ✅ Реализовано

1. **Загрузка документов**
   - Загрузка OpenAPI документов в формате JSON
   - Опциональная привязка к User ID
   - Визуальная обратная связь (успех/ошибка)

2. **Просмотр списка**
   - Отображение всех загруженных документов
   - Информация: ID, описание, дата загрузки
   - Кнопка обновления списка

3. **Чат-интерфейс**
   - Задавание вопросов по выбранному документу
   - Получение ответов от LLM модели
   - История диалога с временными метками
   - Индикаторы загрузки и обработка ошибок

## Быстрый старт

### 1. Установка

```bash
npm install
```

### 2. Настройка

Отредактируйте `.env`:
```env
VITE_API_BASE_URL=http://localhost:8080/api
```

### 3. Запуск

```bash
npm run dev
```

Откройте http://localhost:3000

### 4. Тестирование

Используйте файл `sample-swagger.json` для загрузки тестового документа.

## API Endpoints (Backend)

Приложение ожидает следующие endpoints:

| Метод | Путь | Описание |
|-------|------|----------|
| POST | `/api/documents/upload` | Загрузка документа |
| GET | `/api/documents` | Список документов |
| GET | `/api/documents/{id}` | Получение документа |
| POST | `/api/documents/{id}/chat` | Отправка вопроса |

📄 Полная спецификация в `API_INTEGRATION.md`

### Возможные вопросы и ответы

**Q: Почему не использовали Redux?**  
A: Согласно ТЗ, управление состоянием через React Hooks. Приложение небольшое, Redux избыточен.

**Q: Как обрабатываются ошибки?**  
A: Try-catch блоки вокруг API запросов, отображение через Alert компоненты, возврат состояния при ошибке.

**Q: Почему Vite, а не Create React App?**  
A: Vite быстрее (HMR, сборка), современнее, рекомендуется React документацией.

**Q: Как обеспечивается типобезопасность?**  
A: Strict TypeScript конфигурация, интерфейсы для всех данных, типизация props компонентов.

**Q: Как приложение масштабируется?**  
A: Модульная архитектура, легко добавить новые компоненты/страницы, API слой изолирован.
