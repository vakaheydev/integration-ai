# Руководство по развёртыванию и использованию

## Быстрый старт

### 1. Установка зависимостей

```bash
npm install
```

### 2. Настройка переменных окружения

Создайте файл `.env` (или используйте существующий):

```env
VITE_API_BASE_URL=http://localhost:8080/api
```

### 3. Запуск в режиме разработки

```bash
npm run dev
```

Приложение будет доступно по адресу: http://localhost:3000

### 4. Сборка для продакшена

```bash
npm run build
```

Результат будет в папке `dist/`

### 5. Предпросмотр production сборки

```bash
npm run preview
```

## Использование приложения

### Шаг 1: Загрузка документа

1. На главной странице слева находится форма "Загрузка OpenAPI документа"
2. Опционально введите User ID
3. Нажмите "Выбрать JSON файл" и выберите файл Swagger документа
4. Нажмите "Загрузить документ"
5. После успешной загрузки появится уведомление

**Для тестирования** можно использовать файл `sample-swagger.json` из корня проекта.

### Шаг 2: Просмотр документов

Справа на главной странице отображается список всех загруженных документов с:
- Названием (summary) или ID документа
- Датой загрузки
- Кнопкой обновления списка

### Шаг 3: Работа с документом

1. Кликните на любой документ из списка
2. Откроется интерфейс чата с выбранным документом
3. В текстовом поле внизу введите ваш вопрос по документу
4. Нажмите Enter или кнопку отправки
5. Дождитесь ответа от LLM модели
6. История диалога сохраняется в рамках сессии
7. Для возврата к списку нажмите кнопку "Назад"

## Примеры вопросов для чата

При работе с Swagger документом можно задавать вопросы типа:

- "Какие эндпоинты есть в этом API?"
- "Расскажи про эндпоинт /pets"
- "Какие параметры принимает метод createPet?"
- "Какая структура у объекта Pet?"
- "Какие HTTP методы поддерживает эндпоинт /pets/{petId}?"
- "Опиши схему ответа для GET /pets"

## Особенности реализации

### Управление состоянием

- Используются только React Hooks (useState, useEffect, useCallback)
- Нет глобальных state-менеджеров (Redux, MobX)
- Состояние хранится локально в компонентах

### Обработка ошибок

- Все API запросы обёрнуты в try-catch
- Ошибки отображаются пользователю через Alert компоненты
- При ошибке в чате сообщение пользователя возвращается обратно в поле ввода

### UX улучшения

- Индикаторы загрузки (CircularProgress, состояния кнопок)
- Блокировка элементов во время запросов
- Автоматическая прокрутка чата к последнему сообщению
- Отправка сообщения по Enter (без Shift)
- Multiline поле ввода с поддержкой Shift+Enter для новой строки

### Валидация

- Проверка формата файла (только JSON)
- Проверка заполненности полей перед отправкой
- Валидация на стороне UI перед API запросами

## Интеграция с Backend

Для полноценной работы приложения необходим backend, реализующий API согласно спецификации в `API_INTEGRATION.md`.

**Основные требования к backend:**
1. CORS для localhost:3000
2. Поддержка multipart/form-data для загрузки файлов
3. REST API endpoints (см. API_INTEGRATION.md)
4. UTF-8 кодировка для русскоязычного контента
5. Интеграция с LLM для ответов на вопросы

## Устранение неполадок

### Ошибка при запуске dev сервера

```bash
# Удалите node_modules и переустановите зависимости
Remove-Item -Recurse -Force node_modules
npm install
npm run dev
```

### Ошибки TypeScript

```bash
# Проверьте конфигурацию
cat tsconfig.json

# Убедитесь, что все файлы .tsx имеют правильное расширение
```

### CORS ошибки

Убедитесь, что backend настроен для приёма запросов с `http://localhost:3000`

### Приложение не может подключиться к API

1. Проверьте файл `.env` и значение `VITE_API_BASE_URL`
2. Убедитесь, что backend запущен и доступен
3. Проверьте консоль браузера на наличие ошибок
