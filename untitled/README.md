# OpenAPI Analyzer - Frontend

Frontend-приложение для анализа OpenAPI (Swagger) документов с использованием LLM.

## Технологический стек

- **React 18** - библиотека для создания пользовательского интерфейса
- **TypeScript** - типизированный JavaScript
- **Vite** - быстрый инструмент сборки
- **Material UI (MUI)** - библиотека UI компонентов
- **Axios** - HTTP клиент для работы с API

## Структура проекта

```
src/
├── api/              # Слой взаимодействия с backend
│   └── documentsApi.ts
├── components/       # Переиспользуемые UI компоненты
│   ├── ChatMessageItem.tsx
│   ├── DocumentList.tsx
│   └── UploadDocument.tsx
├── pages/            # Страницы приложения
│   ├── ChatInterface.tsx
│   └── MainPage.tsx
├── models/           # TypeScript интерфейсы и типы
│   └── types.ts
├── App.tsx           # Корневой компонент
└── main.tsx          # Точка входа приложения
```

## Функциональные возможности

### 1. Загрузка документов
- Загрузка OpenAPI документов в формате JSON
- Опциональное указание User ID
- Визуальная обратная связь (уведомления об успехе/ошибках)

### 2. Просмотр списка документов
- Отображение всех загруженных документов
- Информация о каждом документе (ID, описание, дата загрузки)
- Выбор документа для работы

### 3. Чат-интерфейс
- Задавание вопросов по выбранному документу
- Получение ответов от LLM модели
- История диалога с временными метками
- Индикация процесса генерации ответа

## Установка и запуск

### Предварительные требования

- Node.js >= 18.x
- npm или yarn

### Установка зависимостей

```bash
npm install
```

### Настройка переменных окружения

Создайте файл `.env` на основе `.env.example`:

```bash
cp .env.example .env
```

Отредактируйте `.env` и укажите URL вашего backend API:

```
VITE_API_BASE_URL=http://localhost:8080/api
```

### Запуск в режиме разработки

```bash
npm run dev
```

Приложение будет доступно по адресу: http://localhost:3000

### Сборка для продакшена

```bash
npm run build
```

Собранное приложение будет находиться в папке `dist/`.

### Предварительный просмотр production сборки

```bash
npm run preview
```

## API Endpoints

Приложение взаимодействует со следующими endpoints:

- `POST /api/documents/upload` - загрузка документа
- `GET /api/documents` - получение списка документов
- `GET /api/documents/{id}` - получение конкретного документа
- `POST /api/documents/{id}/chat` - отправка вопроса по документу

## Архитектурные решения

### Управление состоянием
Используются React Hooks (useState, useEffect, useCallback) без глобальных state-менеджеров.

### Разделение на слои
- **UI Layer** - компоненты представления
- **API Layer** - логика взаимодействия с backend
- **Models** - типизация данных

### Обработка ошибок
Все API запросы обернуты в try-catch блоки с отображением понятных сообщений пользователю.

### UX улучшения
- Индикаторы загрузки
- Блокировка элементов во время запросов
- Автоматическая прокрутка чата
- Визуальная обратная связь
